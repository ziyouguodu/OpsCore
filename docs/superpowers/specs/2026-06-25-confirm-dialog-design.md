# OpsCore 统一危险操作确认弹窗设计

## 目标

使用统一的应用内确认弹窗替换前端现有 7 处浏览器原生 `window.confirm`，让资产、实例、任务、事件、值班记录、排班模板和用户删除操作保持一致的视觉、交互和反馈。

本轮只实现通用确认弹窗及调用逻辑，不拆分资产台账或其他业务页面。

## 视觉方案

采用已确认的 A 方案：紧凑居中弹窗。

- 暗色背景，与 OpsCore 控制台现有设计体系一致。
- 最大宽度约 420px，移动端保留页面边距。
- 标题区包含清晰的危险操作图标、标题和简短风险状态。
- 内容区展示影响说明和当前操作对象。
- 底部按钮右对齐，取消按钮为普通次级按钮，确认按钮为红色危险按钮。
- 不使用大面积红色背景，保持专业、稳定和高信息密度。

## 组件边界

新增 `frontend/src/components/ConfirmDialog.vue`。

组件只负责展示和交互，不直接调用业务 API。

### 输入属性

- `open: boolean`：控制弹窗是否显示。
- `title: string`：操作标题，例如“删除资产”。
- `message: string`：影响说明，例如“删除后无法恢复，关联凭据也将一并清理。”
- `target: string`：当前操作对象，例如“ASSET-202606-0042 · 支付服务主机”。
- `confirmLabel: string`：确认按钮文案，默认“确认删除”。
- `busy: boolean`：业务请求执行状态。

### 输出事件

- `confirm`：用户确认执行。
- `cancel`：用户取消操作。

组件不负责销毁数据、不访问全局状态，也不持有业务对象。

## 调用模型

`App.vue` 保留一个全局确认状态，并提供 Promise 式调用函数：

```js
const accepted = await requestConfirm({
  title: '删除资产',
  message: '删除后无法恢复，关联凭据也将一并清理。',
  target: `${asset.assetNo} · ${asset.business || asset.hostname || '未命名资产'}`,
  confirmLabel: '确认删除'
})

if (!accepted) return
```

全局状态一次只允许处理一个确认请求。弹窗关闭后必须清理当前 Promise 回调和展示内容，避免后续操作复用旧对象。

## 交互规则

### 打开

- 用户点击删除操作后打开弹窗。
- 弹窗打开时确认按钮自动获得焦点。
- 页面背景增加半透明遮罩并禁止后台误操作。

### 取消

以下操作均视为取消：

- 点击“取消”按钮。
- 点击右上角关闭按钮。
- 点击弹窗外遮罩。
- 按下 `Escape`。

取消后不发起 API 请求，不改变列表、详情或编辑状态。

### 确认

- 点击危险确认按钮后关闭选择阶段，并开始执行原业务删除逻辑。
- `busy` 状态下确认按钮显示“处理中”，取消、关闭和重复确认均不可用。
- API 成功后沿用当前页面已有的列表更新和详情关闭逻辑。
- API 失败后沿用当前全局错误反馈，不错误地从列表移除数据。

### 键盘与无障碍

- 弹窗使用 `role="alertdialog"`。
- 通过 `aria-modal="true"` 表示模态上下文。
- 标题和说明分别通过 `aria-labelledby`、`aria-describedby` 关联。
- 关闭按钮具有“关闭确认弹窗”的可访问名称。
- 弹窗关闭后焦点返回触发删除操作的按钮。

## 覆盖范围

替换以下原生确认逻辑：

1. 删除资产。
2. 删除中间件或数据库实例。
3. 删除任务。
4. 删除事件。
5. 删除值班记录。
6. 删除排班模板。
7. 删除用户。

各调用方使用对应业务名词和对象标识，不使用模糊的统一提示。

## 错误处理

- 没有有效业务对象时不打开弹窗。
- 请求失败时保留原记录和选中状态。
- 请求执行中禁止通过遮罩、关闭按钮或 `Escape` 关闭弹窗。
- 如果页面跳转或用户退出登录时仍有待处理确认，统一按取消处理并清理状态。

## 测试设计

### Playwright 交互测试

新增或扩展一期 UI 巡检，验证：

- 点击资产删除按钮打开应用内弹窗，不出现浏览器原生确认框。
- 弹窗展示正确标题、对象和影响说明。
- 点击取消后弹窗关闭，资产仍存在。
- 按 `Escape` 后弹窗关闭。
- 点击遮罩后弹窗关闭。
- 确认删除后 API 被调用，列表刷新且弹窗关闭。
- 删除请求执行期间确认按钮不可重复点击。

资产场景作为完整确认与删除流程测试；其他六类场景至少验证可以打开统一弹窗并取消，避免每个业务路径重新实现不同交互。

## 非目标

本轮不包含：

- 拆分资产、中间件、任务或事件页面。
- 引入 UI 组件库、状态管理库或路由库。
- 新增后端删除 API 或改变 RBAC。
- 增加批量删除、二次密码验证或审批流程。
- 重构现有 Toast、普通信息弹窗或值班业务弹窗。
